## 一、你在市场上的位置

### 你已经有的（差异化弹药）

| 资产 | 为什么值钱 |
|------|-----------|
| **全链路 ML Pipeline**（StarRocks → 特征工程 → LightGBM → Prefect → MLflow） | 不到一年独立搭建，完整度超过多数 3 年经验的数仓开发 |

## 二、路线：B → C → A

```
现在 ────── B（6-12个月）────── C（1-2年）────── A（2-3年）──→
数据工程师       AI+数据产品        深度学习时序预测
偏ML            智能BI/Text2SQL    Transformer/Nbeats
用AI提效        结合帆软经验        长期护城河
```

### B 阶段：数据工程师（偏 ML）+ AI 提效 → 目标 1.5w+/月

**投递关键词**：「数据工程师」「大数据开发 + 机器学习」「ML 数据管线」

**面试核心故事**：
> 我一年内独立搭建了从 StarRocks 取数到 MLflow 监控的全链路时序预测系统。100+ 维特征工程，LightGBM 模型误差 12%，Prefect 每周自动调度。同时我用 AI 辅助开发，数据工程产出翻倍——省下的时间我用来自动化了周报和建表流程。

**用 AI 提效的具体场景**（面试时要能展开讲）：
- AI 帮你生成复杂 SQL，你怎么验证正确性
- AI 帮你写特征工程代码，你怎么审查看数据流
- AI 帮你调 Spark 参数，你怎么判断建议是否合理

### C 阶段：AI + 数据产品 → 目标 2w+/月

**为什么是自然的延伸**：
- 你有帆软 report 经验，知道业务怎么看数据
- Text-to-SQL 本质是把你的 SQL 能力产品化
- 智能 BI = 你已有的数据工程 + LLM 对话层
- 深圳这类岗位在增长，供给严重不足

**需要提前布局的能力**：
- LLM 应用开发（API、Prompt Engineering、RAG）
- 参考 [ai-engineering-learning-plan.md](ai-engineering-learning-plan.md) 的 Phase 1-4

### A 阶段：深度学习时序预测 → 天花板 3-5w+/月

**为什么放最后**：
- 数学门槛最高，但你有 B/C 阶段积累的数学基础
- 市场对深度学习时序预测的需求稳定（供应链、零售、金融）
- 护城河深——同时懂数仓+特征工程+深度学习的工程师极少

## 四、学习策略：边面边补，面试驱动

```
传统路线：学 6 个月 → 投简历 → 面试 → 发现学的不对 → 再学 → ...
你的路线：投简历 → 面试 → 被问倒 → 精准补 → 再面 → Offer
```

**为什么这样效率高**：
- 你不会花时间学市场不要的东西
- 每次面试失败 = 一份免费的高质量学习清单
- 面多了就知道自己的市场价，谈薪时有底气
- 1.2-1.5w 的岗位你现在就够得着——先拿到一个 offer 再挑

**面试节奏建议**：
- 第 1 周：投 20 家，优先投没那么心仪的，当练手
- 第 2-3 周：每一场面试后立刻复盘，把答不上来的问题记下来
- 第 4 周：开始投心仪的公司

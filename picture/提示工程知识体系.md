## 一、基础概念

### Prompt 是什么
- 给大语言模型的输入指令
- 决定模型输出方向和质量的核心因素

### Token 与上下文窗口
- Token：模型处理的最小语义单元（中文字/词、英文子词）
- Context Window：模型一次能"看到"的最大 token 数量
- 超长上下文的策略：截断、摘要、分块处理

### System Prompt vs User Prompt
- System Prompt：设定角色、规则、边界（优先级最高）
- User Prompt：具体任务指令
- 两者配合形成完整交互

### 关键参数
- Temperature：控制输出的随机性（低=稳定，高=创造性）
- Top-P（核采样）：累积概率阈值过滤
- Top-K：只从概率最高的 K 个 token 中采样
- Max Tokens：限制输出长度
- Stop Sequences：定义停止符号

## 二、提示策略

### 基础策略
- **Zero-shot**：不给示例，直接提问
- **Few-shot**：给 2-5 个示例，让模型模仿格式和风格
- **Instruction Tuning**：明确指令格式

### 思维链（Chain of Thought, CoT）
- 引导模型"一步步思考"
- 零样本 CoT："Let''s think step by step"
- 少样本 CoT：提供带推理过程的示例
- 适用场景：数学推理、逻辑推理、复杂决策

### 高级变体
- **Self-Consistency**：多次采样 + 投票，提高准确率
- **Tree of Thoughts (ToT)**：分支探索多条推理路径，剪枝后选最优
- **Graph of Thoughts (GoT)**：图结构推理，允许多路径合并
- **ReAct**：推理（Reasoning）+ 行动（Acting）交替进行

### 角色扮演
- 设定专业角色（"你是一位资深 Python 工程师"）
- 设定语气和风格
- 提供角色背景知识

## 三、高级技巧

### 提示优化
- 迭代优化：观察输出 → 调整 prompt → 再测试
- Prompt 模板化：变量替换 + 条件分支
- 元提示（Meta-Prompting）：让模型自己优化提示词

### 结构化输出
- JSON 模式：强制模型输出合法 JSON
- XML/Markdown 格式约束
- 使用 Function Calling / Tool Use 获取结构化数据

### 上下文管理
- 对话历史管理（滑动窗口、摘要压缩）
- 长文档处理（RAG、分块检索）
- 多轮对话中的状态保持

### 安全与防御
- Prompt Injection 防御
- 越狱（Jailbreak）防护
- 内容审核过滤

## 四、应用场景

| 场景 | 典型 Prompt 策略 |
|------|-----------------|
| 代码生成 | 明确语言、框架、输入输出格式 |
| 文本摘要 | 指定长度、重点维度、受众 |
| 翻译 | 指定语言对、风格（正式/口语） |
| 分类与标注 | Few-shot + 标签定义 |
| 问答系统 | RAG + 引用来源 |
| 创意写作 | 高 Temperature + 角色设定 |
| 数据分析 | 结构化输出 + CoT 推理 |
| 客服对话 | System Prompt 设定品牌语调 |

## 五、评估与迭代

### 评估方法
- 人工评估：准确性、相关性、流畅度
- 自动评估：BLEU、ROUGE、BERTScore
- LLM-as-Judge：用强模型给弱模型打分
- A/B 测试：两个 prompt 版本对比

### 迭代流程
1. 明确任务和验收标准
2. 写初版 prompt
3. 在小样本上测试
4. 分析失败 case
5. 调整 prompt（改措辞、加约束、换策略）
6. 扩大测试集 → 上线

## 七、最佳实践

1. **明确 > 模糊**：说清楚"做什么、怎么做、输出什么格式"
2. **给示例**：Few-shot 几乎总能提升质量
3. **分解复杂任务**：大任务拆成多步，每步一个 prompt
4. **设定边界**：告诉模型"不要做什么"和"要做什么"同样重要
5. **长 prompt 放前面**：指令放开头，示例放后面
6. **测试边界 case**：不是测试正常输入，是测试模型"翻车"的情况
